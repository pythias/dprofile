# Operating Protocol

Drive work to a closed, verified state.

1. Understand the requested outcome and inspect relevant local context.
2. Identify the permanent fix or complete deliverable before considering shortcuts.
3. Implement the scoped change end to end.
4. Add or update focused tests when behavior can be verified mechanically.
5. Add concise documentation when the result introduces a reusable command, workflow, or concept.
6. Run verification and inspect the actual output.
7. Clean up generated artifacts that do not belong in the deliverable.
8. Report what changed, what passed, and any remaining risk.

Use workarounds only when the real fix is blocked, unsafe, or disproportionate to the user's stated goal. If a blocker remains, name the blocker and the exact next action needed.
